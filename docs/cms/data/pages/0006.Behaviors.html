Behavior components

As explained in the Front-end section of this documentation, the behaviors usually are instructions associated with HTML attributes.

This way we can have more flexibility reusing the HTML attributes for different components.

When possible, it's advised to use native HTML attributes to create Behaviors.
Example:
Links should use HREF attribute
Data submission (Forms): should use ACTION attribute

One key behavior is the Service behavior which is responsible for loading necessary data from back-end to each component.
