Behavior: Service

Selector: '[service]'

The Service Behavior takes care of loading the data and updating the components template.

By adding service attribute to any markup, after the page rendering will start loading the URL passed to it.
<div service="http://something.com/some_service/"></div>
div: {"service":"../api/cms/test/test"}
    span: "#{data}"
It also can be used in conjunction with other HTMLElements behaviors.

For, Sort, Pagination 

Those components if the for attribute is set to the same id attribute of the service HTMLElement, once the service is loaded, it'll update those components.

Events

The Service behavior listens to update event and triggers updated event to itself.

Example:
The Pagination component, when the page is changed, triggers update event to the Service HTMLElement.
Service will load the data, and once load is complete it triggers update to all elements referencing Service HTMLElement and also triggers updated event to itself, so components listening to updated event in it's HTMLElement will be able to control it's structure.
