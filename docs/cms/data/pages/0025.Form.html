Form

The form component is a Form HTML Markup.
It uses a usual Form markup attributes such as action and method.
You can also set success attribute to tell what to do when the request is successful.
Success attribute can be one of below:

refresh: Will re-render the current page
reload: Will reload the entire window (Browser refresh)
URI of CMS page: will change the CMS page.

There's a special attribute type which you can set what data type you will send.
For now only json and normal is supported. Setting to json will serialize all data in form to a JSON format according to the input attribute json-name building it's hierarchy and submit it.

<form type="json">
	<div json-name"test">
		<input name="name" value="1" />
	</div>
</form>

Also, have a look at API class, it may give you better understanding on what happens behind the scene.

Inputs

The inputs usually should be grouped inside Field tag and have Label tag inside it.
<field>
<label>Some label</label>
<input type="text" name="name" />
</field>
Inside the Field tag can also contain Validation to validate the data in the input value.
