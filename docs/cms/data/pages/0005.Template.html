Mara template system

Mara is a template system created to create HTML markups using dynamic data.
The name comes from a demon: https://en.wikipedia.org/wiki/Mara_(demon)
Note: The Mara template is not fully implemented in PHP, although it suffice to render the initial structure of CMS.

Syntax

The syntax is indentation based, resembles YAML and JSON with some tweaks.

Here's the example of the syntax.
# Root template
html:
	body:
		h1: #{title}
		h2: "Description"
		div:{"_": "a", "id":"Some id", "title": "#{title}"}
		div.className:
		div#id:
		(#{items})
			div#id.className:
				test: #{items}
					"Loop me"
		>otherstructure
		>iteratestructure: #{items}
		>iteratestructure2: #{items}

		@test/test: #{items}

<otherstructure
	div: "SOME STRUCTURE"

<iteratestructure
	div: #{}
<iteratestructure2
	div: #{value}

Indentation

The indentation is used to create markup hierarchy.
Every following indented block is a child markup.
----MARA CODE----
html:
    body:


----RENDERED HTML MARKUP----
<html>
    <body></body>
</html>
Every line (or instruction blocks) has the following structure.

[Decorator][name][. CSS className][# element ID]: [html attributes in JSON format] [associated data object]

Decorator tells what each block does. If it's not present, it will render as HTML markup.
Name is the markup tag name or in case of instruction, the reference name.
Following the markup name, the dot (.) and sharp (#) can be used to add multiple class names and ids.
The HTML attributes should be in JSON format.
Associated data object, tells what object should be used to render the following block. If it's inside quotes (") it'll be used as a JSON and therefore it's a HTML attribute.

Decorators

There are some decorators to make the template modular.
<: create a independent block
>: loads the independent block
@: loads a external .mara template file
@path/to/template>reference: loads the external template file and uses the reference block.

Variables

The variables are referenced using #{name} notation.

Conditionals

The conditionals are encapsulated in ().
It uses JavaScript eval() function to execute the condition and uses the local object to retrieve the variables.

Loops

When rendering the blocks, Mara will assume all data is a Array, this way, if there's actually a Array in the Object, it'll loop through each item and render the blocks following.
